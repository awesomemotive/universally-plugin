# Universally — WordPress Plugin

Automatic website translation and localization for WordPress. Translates pages into 110+ languages with SEO-friendly URLs and hreflang tags.

This repository hosts the public, human-readable source for the [Universally Language Translation Multilingual Tool](https://wordpress.org/plugins/universally-language-translation-multilingual-tool/) plugin. The compiled JavaScript and CSS shipped to wordpress.org are generated from the TypeScript/SCSS sources in `plugin/panel/js/`.

## Layout

```
plugin/                  # plugin source (ships to wp.org)
  universally.php        # plugin bootstrap
  app/                   # PHP classes (PSR-4, `Universally\` namespace)
  includes/              # procedural helpers, REST entry, conflict handler
  panel/js/              # TypeScript/React sources for the admin panel
  panel/build/           # compiled JS/CSS (generated by `npm run build`)
  panel/src/             # PHP-side panel framework
  assets/                # static plugin assets (icons, switcher JS, etc.)
  languages/             # translations and POT file
  readme.txt             # wp.org listing
assets/                  # wp.org plugin page assets (banners, icons, screenshots)
scripts/release.sh       # release entry — bumps, builds, tags, pushes
docker-compose.yml       # local WordPress for quick testing
```

## Local development

Node.js 18+ and PHP 7.4+ are required.

```sh
cd plugin
npm install
npm run build       # production build
npm run start       # watch mode
composer install    # PHP autoloader
```

`npm run build` regenerates everything inside `plugin/panel/build/` (compiled JS, CSS, source maps, and `*.asset.php` dependency files) from the sources in `plugin/panel/js/` using [@wordpress/scripts](https://www.npmjs.com/package/@wordpress/scripts).

## Test in a real WordPress

```sh
docker compose up -d
open http://localhost:8765
```

Run the installer, then activate the plugin from `Plugins → Installed Plugins`. The plugin directory is bind-mounted live, so changes to PHP take effect immediately; changes to TS/SCSS require `npm run build` (or `npm run start` for watch mode).

```sh
docker compose down -v   # wipe DB + WP install when done
```

## Releasing

```sh
npm run release
```

Interactive menu:

- `1) patch` — `1.0.0 → 1.0.1`
- `2) minor` — `1.0.0 → 1.1.0`
- `3) major` — `1.0.0 → 2.0.0` (extra confirmation)
- `4) beta` — next `vX.Y.Z-beta.N` for the unshipped stable
- `5) rc` — next `vX.Y.Z-rc.N`
- `6) custom` — enter any semver string
- `7) rebuild current` — force re-release of the current version (replaces the tag)

The script bumps `plugin/universally.php`, `plugin/package.json`, and `plugin/readme.txt` (`Stable tag` for stable releases), commits, creates an annotated `vX.Y.Z` tag, and pushes.

CI (`.github/workflows/release.yml`) picks up the tag push:

- **Stable tag** (`vX.Y.Z`) → builds, deploys to wp.org SVN trunk + `tags/X.Y.Z` + `assets/` via [10up/action-wordpress-plugin-deploy](https://github.com/10up/action-wordpress-plugin-deploy), creates a GitHub Release with the zip + a one-click WordPress Playground link.
- **Pre-release tag** (`vX.Y.Z-beta.N` / `-rc.N`) → builds, creates a GitHub Release (marked pre-release) with the zip + Playground link. Skips SVN.

Required repo secrets: `SVN_USERNAME` and `SVN_PASSWORD` (wp.org account credentials).

## Translations

POT regeneration is normally done in CI during the build. To regenerate locally (Docker required):

```sh
cd plugin
npm run makepot
```

## Bundled third-party libraries

The compiled JS in `plugin/panel/build/index.js` bundles:

- [@timkit/conditions](https://www.npmjs.com/package/@timkit/conditions) (Apache-2.0) — condition evaluator for dynamic field visibility.

All other JS dependencies (React, `@wordpress/*`) are externalized at build time and resolved to the script handles WordPress already provides.
