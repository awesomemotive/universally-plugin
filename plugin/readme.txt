= Translate Your WordPress Site into 110+ Languages =

Most WordPress translation plugins ask you to duplicate every post, every product, and every menu — then translate strings one at a time inside the admin. Universally does the opposite. The plugin sits between your visitor and your page, translates the full HTML output on the fly, and serves a fully translated page back. Whether your content is built with the block editor, a page builder, a shortcode, or a custom theme, the translation just works.

You don't translate anything manually. Universally detects new content automatically — posts, pages, products, menus, widgets, custom fields, and dynamic content — and translates it as it appears. Publish a new blog post or add a new WooCommerce product, and the translation is generated on the next visit and cached for everyone after. Translate once, serve forever.

= Multilingual SEO with Hreflang, Translated URLs, and Translated Metadata =

[Multilingual SEO](https://universally.com/multilingual-seo/?utm_source=wprepo&utm_medium=link&utm_campaign=liteplugin) is the whole reason you translate your site. If Google can't index your translated pages, you have a translated site that nobody finds. Universally handles every technical multilingual SEO requirement automatically.

Each language gets its own URL prefix (`/fr/`, `/es/`, `/de/`) so each translation is fully crawlable and indexable. Hreflang tags are added to every translated page so Google serves the right language to the right country. Page titles, meta descriptions, Open Graph tags, and Twitter card metadata are translated for an optimized snippet in every language. XML sitemaps include every translation, canonical tags point correctly, and translated content gets indexed as translated content. Universally is fully compatible with All in One SEO, Yoast SEO, Rank Math, and SEOPress — your existing SEO settings carry over to every translated page.

= Custom Translation Rules for Granular Multilingual Control =

Sometimes the right translation is no translation at all — like a code block, a legal disclaimer that must appear in the source language, or a section of a page where translation breaks the layout. Universally [custom translation rules](https://universally.com/custom-translation-rules/?utm_source=wprepo&utm_medium=link&utm_campaign=liteplugin) let you exclude specific selectors, classes, IDs, or entire pages from translation with zero code.

Want to keep the pricing table in English but translate the rest of the page? Add a rule. Need to translate everything except a specific shortcode output? Add a rule. Custom translation rules give you precise control over what Universally translates and what it leaves alone, all from a visual rule editor in your dashboard.

= External Service =

This plugin connects to the [Universally](https://universally.com/?utm_source=wprepo&utm_medium=link&utm_campaign=liteplugin) translation service to process translations. When a visitor requests a translated page, the page content is sent to Universally servers for translation and the translated HTML is returned to the visitor.

The page source code is not stored. Universally only stores the individual translated strings so they can be reused on future visits without re-translating. Plugin settings let you exclude specific strings, selectors, or pages from translation, so anything you don't want sent to the service can be opted out at the source.

== Installation ==

1. Install Universally from the WordPress plugin directory, or upload the `universally-language-translation-multilingual-tool` folder to `/wp-content/plugins/`.
2. Activate the plugin through the **Plugins** menu in WordPress.
3. Go to the Universally settings page and enter your API key. You can get a free API key at [universally.com/pricing/](https://universally.com/pricing/?utm_source=wprepo&utm_medium=link&utm_campaign=liteplugin).
4. Pick your target languages in the Universally dashboard.
5. Your WordPress site is now multilingual. Visit `yoursite.com/fr/` (or any language code you selected) to see your first translated page.

= Do I need any coding skills to use this translation plugin? =

No. Universally works out of the box. Install the plugin, enter your API key, pick your languages, and your WordPress site is automatically translated into every language you select. No coding, no template editing, no shortcodes required — although shortcodes, Gutenberg blocks, and a PHP function are available if you want to place the language switcher manually.

= How does the AI translation work? =

When a visitor requests a translated page (for example, `yoursite.com/fr/about/`), Universally translates the full page HTML using AI and returns the translated page. Translations are cached on a global edge network so subsequent visits are served instantly with no extra translation cost.

Universally supports 110+ languages, including Spanish, French, German, Italian, Portuguese, Brazilian Portuguese, Dutch, Polish, Russian, Ukrainian, Turkish, Arabic, Hebrew, Hindi, Bengali, Urdu, Chinese (Simplified and Traditional), Japanese, Korean, Vietnamese, Thai, Indonesian, Malay, Filipino, Swahili, and many more. = Does the translation plugin work with WooCommerce? =

Yes. Universally translates everything visitors see, including WooCommerce product pages, product categories, the cart, the checkout, my-account pages, and order confirmation emails. Because translation happens at the HTML level, WooCommerce translation works out of the box with no per-product configuration.

= Does it work with Elementor, Divi, and other page builders? =

Yes. Universally is compatible with Elementor, Divi, Beaver Builder, WPBakery, SeedProd, Gutenberg, Brizy, Oxygen, Bricks, and any other page builder. Translation happens at the output level, so it doesn't matter how the content is built.

= Is the translation plugin SEO friendly? =

Yes. Every language gets its own URL prefix (`/fr/`, `/es/`, `/de/`), hreflang tags are added automatically to every page, page metadata is translated for each language, and translated pages appear in multilingual XML sitemaps. Search engines crawl and index each translation as a separate page. Universally is fully compatible with All in One SEO, Yoast SEO, Rank Math, and SEOPress.

= Can I customize the language switcher? =

Yes. You can use auto-placement (bottom-right, bottom-left, top-right, top-left) or place the language switcher manually with a shortcode, Gutenberg block, or PHP function. Translations are processed and cached on Universally's servers. The translated strings are stored so they can be reused on every future visit without re-translating. No additional database tables and no translation row bloat on your WordPress installation. Your WordPress database stays exactly the size it was before you installed the multilingual plugin.

= Is there a free translation plan? =

= Will the translation plugin slow my site down? =

No. Universally translates pages in the cloud and serves them from a global edge network. Translated pages are cached, so visitors get the same fast load times you have on your source language. There's no database bloat, no extra database queries per page, and no heavyweight admin process scanning your content for strings.

= Does it work with caching plugins and CDNs? =

Yes. Universally is compatible with WP Rocket, W3 Total Cache, WP Super Cache, LiteSpeed Cache, Cloudflare, BunnyCDN, and other caching plugins and CDNs. Translated pages are cacheable just like the source language version.

Yes. Universally fully supports RTL languages, including Arabic, Hebrew, Persian, Urdu, and Pashto. Page direction, alignment, and layout flip automatically on RTL translations.
